Kubernetes & Orchestration

Mastering Kubernetes Commands & Best Practices

Kubernetes (K8s) is an open-source container orchestration platform that automates the deployment, scaling, and management of containerized applications. It provides essential tools to manage microservices architectures, offering features like service discovery, load balancing, self-healing, and automated rollouts and rollbacks.

Core Concepts:

  • Pod: The smallest deployable unit in Kubernetes, representing one or more containers.
  • Service: Defines a logical set of Pods and a policy by which to access them.
  • Deployment: Manages the creation and scaling of Pods, ensuring they stay running.
  • Namespace: A way to divide resources in a Kubernetes cluster, improving organization and security.
  • Node: A worker machine in Kubernetes that runs containerized applications.

Essential Kubernetes Commands:

  • kubectl get pods - List all running pods in the current namespace, showing their status.
  • kubectl describe pod <pod_name> - Show detailed information about a specific pod, including events and conditions.
  • kubectl apply -f <file>.yaml - Apply a YAML configuration file to create or update resources like Pods, Deployments, and Services.
  • kubectl delete pod <pod_name> - Remove a specific pod from the cluster.
  • kubectl get services - Display all services running in the cluster, showing IPs and exposed ports.

Advanced Kubernetes Commands:

  • kubectl rollout restart deployment <deployment_name> - Restart a deployment to update all Pods with the latest configuration.
  • kubectl get nodes - List all nodes in the cluster, showing their status, resources, and conditions.
  • kubectl exec -it <pod_name> -- /bin/bash - Start a shell session inside a running container in a pod, useful for debugging.
  • kubectl logs <pod_name> - Retrieve the logs of a specific pod for debugging purposes.
  • kubectl top nodes - Display resource (CPU/memory) usage statistics for each node.
  • kubectl top pods - Show resource usage of all running pods in the cluster.
  • kubectl expose deployment <deployment_name> --type=NodePort --port=8080 - Expose a deployment as a service on a specific port.

Scaling & Managing Kubernetes:

  • kubectl scale deployment <deployment_name> --replicas=3 - Scale the number of replicas (pods) for a deployment.
  • kubectl autoscale deployment <deployment_name> --min=1 --max=10 --cpu-percent=80 - Set up autoscaling based on CPU utilization.
  • kubectl delete deployment <deployment_name> - Delete a deployment, removing all its pods.

Practice Sets:

  • Basic: Create a new pod and expose it via a service, then inspect logs and debug the pod.
  • Intermediate: Create a deployment with multiple replicas and use autoscaling based on CPU utilization.
  • Advanced: Configure a multi-container pod with sidecar patterns, set up secrets, and use a config map for environment variables.

Best Practices for Kubernetes:

  • Use Namespaces: Organize resources by teams or projects to isolate environments and control access.
  • Keep Configurations as Code: Store YAML files in version control to track changes and rollback easily.
  • Set Resource Limits: Define CPU and memory limits for each pod to prevent resource exhaustion.
  • Monitor and Observe: Use tools like Prometheus, Grafana, and Kubernetes dashboard to keep an eye on resource usage and application health.
  • Use Rolling Updates: Ensure zero downtime by using Kubernetes rolling updates during deployments.
  • Leverage Helm: Use Helm charts to deploy complex applications with a single command.
